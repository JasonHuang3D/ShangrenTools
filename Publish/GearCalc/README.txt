主要用途：
    1) 输入自己当前整个仙界的所有真实信息，此模拟器的用途就是找到最优方案，包括产晶，各战力，总战力等。
    2) 输入未来可能有的信息(虚拟仙人基础属性，虚拟仙器，虚拟仙职，虚拟头像等)，此模拟器的用途就是模拟未来的仙界属性，包括产晶，各战力，总战力等。

可配置文件：
    XianjieData.json 文件
        -此文件是仙界所有信息，包括：头像，仙侣，仙职，仙人，产业。具体请查看该文件内的说明。
    XianqiData.json 文件
        -此文件是仙器信息，具体请查看该文件内的说明。
    
    注意：
        -推荐使用 Visual Studio Code 进行编辑上述json文件，会自动检查是否存在语法错误等。https://code.visualstudio.com/
        -作者已将自己的范例数据给出，请根据自己情况更改。
        -目前版本，除产业节点对于名字和数量（数量只能少不能多）有严格限制，其他所有节点均可以自行添加，如仙职，仙侣，头像，仙人。
         且对节点名字不限。之后会将产业计算公式开放到json文件里，届时所以节点数量和名称都不会受到限制。

操作及算法说明：
    对话框输入: 
        数字1：此模式计算"最佳仙人总属性"。
            -从给出仙人里挑选可装配数量的仙器，从而达到所给出仙人的力 念 福 3属性的总和最大，不受游戏词条:"宗门总..."的影响。
            -比如： 
                1)  给出"仙人1 仙人2 仙人3",以及40件仙器挑选7件可装备的仙器，那么最后的结果为 
                    穿戴挑选出来的7件装备后, 仙人1的最终力念福之和 + 仙人2的最终力念福之和 + 仙人3的最终力念福之和，并这个和最大。
                2)  给出"仙人1",那么最终结果就只是仙人1的的最终力念之和，并使其最大。
            -适用场景:
                -计算天元副本及虚灭副本的最优的配装方案(如 力仙，念仙配装)。
                -模拟洗练结果或未获得的装备对于当前某个或某组仙人的影响，可将其当做一个仙器文件输入，
                 来看其对于仙人的影响或者是否能替换现在的装备等。
                -模拟仙人升级，模拟仙职升级，模拟头像解锁，模拟仙侣升级，模拟...

        数字2：此模式计算"最佳宗门总力念"。
            -从给出仙人里挑选可装配数量的仙器，从而达到所给出仙人的力 念 2属性的总和最大，受游戏词条:"宗门总..."影响。
            -比如： 
                1)  给出"仙人1 仙人2 仙人3...",以及40件仙器挑选7件可装备的仙器，那么最后的结果为 
                    穿戴挑选出来的7件装备后，游戏里宗门页面里的宗门总仙力和宗门总仙念之和最大。
            -适用场景:
                -计算仙界幻境及仙界推图的最优的配装方案。
                -模拟洗练结果或未获得的装备对于当前某个或某组仙人的影响，可将其当做一个仙器文件输入，
                 来看其对于仙人的影响或者是否能替换现在的装备等。
                -模拟仙人升级，模拟仙职升级，模拟头像解锁，模拟仙侣升级，模拟...

        数字3：此模式计算"最佳产仙晶"。
            -从给出仙人里挑选可装配数量的仙器，以及对给出仙人进行挑选以及安排产业的站位，从而达到产晶每轮最大化。
            -比如： 
                1)  给出"仙人1 仙人2 仙人3...", 40件仙器挑选7件可装备的仙器，提供5个产晶产业（可以小于5个），15个站位，那么最后的结果为 
                    穿戴挑选出来的7件装备后，以及挑选了15个仙人，并且安排这15个仙人的产业站位，从而达到最大产晶。
            -适用场景:
                -计算仙界产晶的最优的方案，包括仙人挑选，仙人站位，装备选择。
                -模拟洗练结果或未获得的装备对于当前产晶的影响，可将其当做一个仙器文件输入，
                 来看其对于仙人的影响或者是否能替换现在的装备等。
                -模拟仙人升级，模拟仙职升级，模拟头像解锁，模拟仙侣升级，模拟...

        数字4：此模式计算"最佳产仙能"。
            -和第三个模式产晶一样，产仙晶换成产仙能，请参照上文。

        q: 退出。
        c: 清屏。

    注意：
        -算法精度取决于仙人的基础属性精度，如果输入的基础属性精确到个位那么结果就会精确到个位，到亿位就精确到亿位，以此类推。
        -提高精度:
            -方法如自己通过仙界面板仙人列表的个位数的最终属性，自行反推一遍(如去除装备，去除头像，去除仙侣天赋)。
            -后续如游戏如不增加基础属性个位数显示，会考虑增加模式5，用来辅助反推基础属性。
        -所有算法模式:
            -从XianjieData.json文件中的节点"参与运算仙人"中选取仙人。
            -从XianqiData.json文件中的节点"参与运算仙器"中选取仙器。
        -无需关闭程序，也可以更改外部配置文件。 

算法性能：
    -主要取决于参与运算的仙器的数量，和可装备仙器数量。比如 43个仙器里面选择7个，那么就有C43,7种可能性 = 32,224,114. 每一种可能性和所有仙人（如35个）进行计算。
     所以，总共需要计算 32,224,114 * 35次 = 112,843,990次。
    -减少装备个数能有效提高性能，上述例子，i78700k CPU需要约21秒。
